Predicting Fatality of Terrorist Attacks¶

Cailin O'Toole, Bethany Hsiao¶

Introduction¶

Thousands die every year from terrorist attacks and millions more have their lives affected by terrorism. It is incredibly important to understand what makes a terrorist attack deadly so we can work to mitigate this risk as a global community. In this tutorial we will attempt to discern trends in terrorist attacks to determine the variables that make the attack likely to cause fatalities. This data can be used to decide which regions need the most assistance fighting terrorism and which types of attacks more resources should go towards preventing.The Global Terrorism Database (GTD) contains data on over 100,000 terrorist attacks, collected by the START center in partnership with the University of Maryland. This data will allow us to predict which attacks will be deadly.

The GTD chooses to include terrorist events as terrorist attacks if they meet two of three criteria.

1) The attack is aimed at attaining a political, economic, religious, or social goal (GTD)and is not done exclusively for profit.

2) The attack is attempting to coerce, intimidate, or convey some other message to a larger audience (GTD) beyond just the attack victims.

3) The attack is not a part of legitimate warfare activities (GTD) because it targets noncombatants.

It must be noted that the GTD does not account for cyberattacks because the complexity that including these types of attacks would require, however GTD along with start is working to develop the ability to track cyber attacks. To learn more about the Global Terrorism Database team and the mission of the National Consortium for the Study of Terrorism and Responses to Terrorism (START) visit their webpage.

Data Collection¶

We will use the pandas python libraries to work with our data. Click here for more information on installing these libraries and here for python documentation.

The Data Lifecycle (https://www.usgs.gov/products/data-and-tools/data-management/data-lifecycle) begins at data collection (or data acquisition). For this we will be using the GTD dataset, which can be imported here (https://www.start.umd.edu/gtd/access/) as a csv file. We will read these csv files into a DataFrame, which allows us to visualize data as a matrix with rows and columns. We can manipulate this matrix by iterating through the rows and columns or by deleting or combining rows and columns.

Although the GTD has data on terrorist attacks going back to the 1970s, we will choose only to look at a smaller subset of data. Technological improvements and the growth of the internet has greatly improved collection techniques. The GTD warns that data may appear to show differences in attack level between 1998, 2008, and 2012 because of changes in data collection methodology. In order to work with a more standardized data set we will only look at data since 2012. 2012 marks the date that data collection for the GTD was moved to the University of Maryland. To read more about the GTD dataset click here.

Tidying and Modifying the Data¶

To clean the data and focus only on variables that might affect the fatality of an attack, we removed many columns from the database. Some columns like attackType and weaptype2 were removed because they track the presence of a secondary mode of attack or secondary weapon and most of these variables are left empty because there was only a single type of weapon or mode of attack in the terrorist incident. We felt other columns like resolution, which is the date an extended incident was resolved, did not have a strong bearing on whether or not there were fatalities in an attack. Finally we removed several columns like provstate or addnotes because they contained unique comments for each attack that were unrepeated.

We are left with 31 columns in the dataset. Most of the column names explain themselves but a few explanations have been added. To learn more about the dataset codebook visit. (https://www.start.umd.edu/gtd/downloads/Codebook.pdf)

First, we are going to modify the nkill variable to turn it from a numerical variable to a new categorical variable called Killed denoting whether or not there were fatalities. If the value of nkill is 0 we will enter 0 in Killed, but if the value of nkill is any other number we will place a 1 in killed. This will allow us to perform machine learning.

Many of the columns contain values like unknown or -99 to indicate uncertainty or lack of information. Because these incidents are not properly reported we will remove them from our dataset.

In order to check if these variables are also able to predict if an attack will have a casualty, rather than just a fatality we will create an additional casualty column. If there have been any fatalities OR any injuries the number in the casualty column will be 1, otherwise it will be 0.

Finally, we used the dropna() function which is part of the pandas library. The dropna() function drops any NA values. There are many empty values in the GTD dataset and when transforming the csv files into the pandas dataset they are automatically transformed into an NA, so dropna will remove any incomplete lines for us.

Data Exploration¶

Before we begin trying to predict whether or not an attack will cause fatalities, we are going to explore the data to get a better understanding of what the data looks like. The GTD gives us many different variables that affect the type of attack and we can look for trends by observing some of them individually.

We will now create a map of the world, populated with number of kills by longitude and latitude to visualize how many terrorist attacks occurred in which regions. This visualization is presented using a Folium HeatMap (which you can learn more about here and allows us to see clearly which countries are targets of fatal terrorist actions the most often. Red represents the least attack while blue represents the most. We are able to see that the most attacks are concentrated in regions experiencing unrest like Colombia, Pakistan, parts of Africa, and the Middle East.

Unfortunately this does not show an immediate clear line of regression, but we do notice that there appears to be a clustering of data around 40, 30, 25, 20, 15 attackers. This is possibly because a weakness in the dataset and a weakness in data collection. In terrorist attacks it is difficult to determine the exact number of assailants involved so media and law enforcement are often forced to estimate, leading to a disproportionate amount of data clustered around those values. We dont see the same effect when considering the number of fatalities because victims are often left behind and law enforcement officials can get an exact count.

Because some regions have better health care and more effective and fast responding law enforcement, fatality rates might be much different according to regions. Here we do a chi-squared test to check for a relationship between suicide attacks and attacks with fatalities. Because suicide attacks mean that an attacker kills without regard to their own life, we hypothesize that suicide attacks are more likely to have fatalities than non-suicide attacks. In the example of a suicide bomber, intending to sacrifice oneself in a terrorist bombing means the attacker can choose in the moment where to set off the bomb to cause the most damage, making suicide attacks more likely to result in deaths. To do a chi-squared test we start with the null hypothesis that there is no difference in likelihood between each of the four sections, non-suicide non-fatal, suicide fatal, suicide non-fatal, and non-suicide fatal, meaning each should happen a roughly equal number of times. Our critical value is .95 meaning we can reject the null hypothesis with 95% certainty if the p-value is less than .05. The returned p-value is less than .05 so we reject the null hypothesis, meaning the variable Suicide and Killed are dependent and we should include suicide in our machine learning as a variable that influences whether an attack is fatal.

Conclusion¶

Our prediction for whether or not an attack would be deadly could use some improvement, but it is a start. While variables like weapon type, attack type, whether it was a suicide attack, and the number of attackers do have some bearing on whether an attack will be fatal, there are many other factors at play. Additionally there are flaws in the data as a result of the dangerous environment terrorist attacks cause, and because in many regions were terrorism occurs institutions are weak, disorganized, and corrupt, and have difficulty obtaining accurate supports. Another weakness in the dataset is that countries may want to hide terrorism within their borders and fail to report it or even cover it up. Another potential variable that was included in the dataset but had too many unique identifiers for our purpose was the Group name variable. This could have significant impact onto the deadliness of attacks because different groups have different modus operandi and different training. Future work should include this variable as well. Although the nature of terrorism makes the GTD imperfect, it is still an incredible wealth of knowledge that can be used to improving lives worldwide. Future data science investigation into the topic could lead to new U.S. foreign policies or treatises of behavior abroad.
